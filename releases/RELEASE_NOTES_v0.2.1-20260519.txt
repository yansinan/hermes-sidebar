HIGHLIGHTS
----------

This release upgrades Hermes Sidebar to a Runs-first execution model and
aligns process feedback across the sidepanel and right-click workflow.

✓ Runs API is now the primary transport, with Chat streaming fallback when needed
✓ Process updates are unified into a single blue process bar
✓ Timeline events are persisted and shown consistently in both sidepanel and right-click flows
✓ Hermes Runs SSE parsing now follows the documented event model more closely
✓ Session reuse is improved so session ids are carried forward across requests

WHAT CHANGED
------------

Transport and event handling

- Added Runs-first request orchestration with Chat fallback
- Extended Chat lifecycle feedback to include thinking, streaming start, and completion events
- Added support for comment-based SSE markers such as `: keepalive` and `: stream closed`
- Modeled `waiting_for_approval` end-to-end in the client, controller, and UI
- Centralized Runs status wording in a shared enum table for consistent labeling

UI and conversation behavior

- Unified live progress into the process bar instead of duplicating placeholder assistant bubbles
- Suppressed empty assistant streaming rows until there is real content or tool output
- Added timeline event injection for extraction and right-click flows
- Reused shared process/timeline helpers across sidepanel and background workflows
- Refined timeline visuals with lighter tags and optional detail expansion

Stability and integration

- Improved reuse of captured `session_id` / server session references
- Added targeted regression coverage for Runs parsing and controller transport behavior
- Verified build and targeted tests after the refactor
